Activity 1

Comments are explanatory notes that programmers add to their code to describe what the code is doing or why it is doing it. These notes are not executed by the computer, but serve as documentation for other programmers who may need to read and understand the code.

Activity 2

Async/await is a feature in programming languages that allows developers to write asynchronous code in a more synchronous style. Asynchronous code does not block program execution while waiting for an operation to complete. By using specific syntax and keywords, async/await enables developers to write asynchronous code that looks and behaves more like synchronous code. This makes it easier to understand and maintain.

Activity 3

API documentation is a collection of materials that explains how to interact with an Application Programming Interface (API). It includes details about the API's features, such as endpoints, formats for requests and responses, error codes, and requirements for authentication and authorization. API documentation provides developers with a reference guide for integrating an API into their applications or software. It helps developers to quickly understand the API's capabilities and how to use it effectively. The documentation should be well-organized, easy to navigate, and include examples and sample code. API documentation can be presented in different formats, such as HTML, PDF, or interactive documentation that allows developers to test the API directly from the documentation.
